## ⚙️ 預設設置說明（重要！）

預設採用以下優化策略，主要是為了節約 AI token 消耗：

| 預設設置       | 說明                                    | 如何調整                              |
| -------------- | --------------------------------------- | ------------------------------------- |
| **限制條數**   | 預設返回 50 條新聞                      | 對話中說"返回前 10 條"或"給我 100 條" |
| **時間範圍**   | 預設查詢今天的數據                      | 說"查詢昨天"、"最近一周"或"1月1日到7日" |
| **URL 連結**   | 預設不返回連結（節省約 160 tokens/條）  | 說"需要連結"或"包含 URL"              |
| **關鍵詞列表** | 預設不使用 frequency_words.txt 過濾新聞 | 只有調用"趨勢話題"工具時才使用        |

**⚠️ 重要：** AI 模型的選擇直接影響工具調用效果，AI 越智能，調用越準確。當你解除上面的限制，比如從今天的查詢，放寬到一周的查詢，首先你要在本地有一周的數據，其次，token 消耗量可能會倍增。

**💡 提示：** 本專案提供了專門的日期解析工具，可以準確解析"最近7天"、"本周"等自然語言日期表達式，確保所有 AI 模型獲得一致的日期範圍。詳見下方 Q18。

## 基礎查詢

### Q1: 如何查看最新的新聞？

**你可以這樣問：**

- "給我看看最新的新聞"
- "查詢今天的熱點新聞"
- "獲取知乎和微博的最新 10 條新聞"
- "查看最新新聞，需要包含連結"

**工具返回行為：**

- 工具會返回所有平臺的最新 50 條新聞
- 預設不包含 URL 連結（節省 token）

**AI 展示行為（重要）：**

- ⚠️ **AI 通常會自動總結**，只展示部分新聞（如 TOP 10-20 條）
- ✅ 如果你想看全部 50 條，需要明確要求："展示所有新聞"或"完整列出所有 50 條"
- 💡 這是 AI 模型的自然行為，不是工具的限制

**可以調整：**

- 指定平臺：如"只看知乎的"
- 調整數量：如"返回前 20 條"
- 包含連結：如"需要連結"
- **要求完整展示**：如"展示全部，不要總結"

### Q2: 如何查詢特定日期的新聞？

**你可以這樣問：**

- "查詢昨天的新聞"

**工具返回行為：**

- 不指定日期時自動查詢今天（節省 token）
- 工具會返回所有平臺的 50 條新聞
- 預設不包含 URL 連結

**AI 展示行為（重要）：**

- ⚠️ **AI 通常會自動總結**，只展示部分新聞（如 TOP 10-20 條）
- ✅ 如果你想看全部，需要明確要求："展示所有新聞，不要總結"

### Q4.4: 如何同時搜索熱榜和 RSS 內容？

**你可以這樣問：**

- "搜索'AI'相關內容，包括 RSS"
- "查找'人工智慧'的新聞，同時搜索 RSS 訂閱"
- "搜索'特斯拉'，熱榜和 RSS 都要"

**工具返回行為：**

- 熱榜結果和 RSS 結果**分開展示**
- 熱榜按排名/相關度排序，RSS 按發布時間排序
- RSS 結果不影響熱榜的排名展示
- 預設返回熱榜 50 條 + RSS 20 條

**可以調整：**

- RSS 數量：如"RSS 返回 10 條"
- 只搜索熱榜：不說"包括 RSS"（預設行為）
- 只搜索 RSS：說"只在 RSS 中搜索"

**你可以這樣問：**

- "分析一下今天新聞的情感傾向"
- "看看'特斯拉'相關新聞是正面還是負面的"
- "分析各平臺對'人工智慧'的情感態度"
- "看看'比特幣'一周內的情感傾向，選擇前 20 條最重要的"

**工具返回行為：**

- 預設分析今天的數據

- 按權重排序（優先展示重要新聞）
- 預設不包含 URL 連結

**AI 展示行為（重要）：**

- ⚠️ 本工具返回 **AI 提示詞**，不是直接的情感分析結果
- AI 會根據提示詞生成情感分析報告
- 通常會展示情感分布、關鍵發現和代表性新聞

**可以調整：**

- 指定話題：如"關於'特斯拉'"
- 指定時間：如"最近一周"
- 調整數量：如"返回前 20 條"

### Q9: 如何獲取去重後的跨平臺新聞？

**你可以這樣問：**

- "幫我聚合今天的新聞，去掉重複的"
- "看看哪些新聞在多個平臺都有報導"
- "給我看去重後的熱點新聞"
- "哪些新聞是跨平臺熱點"

**工具功能：**

- 自動識別不同平臺報導的同一事件
- 將相似新聞合併為一條聚合新聞
- 顯示每條新聞的平臺覆蓋情況
- 計算綜合熱度權重

**返回信息：**

| 欄位 | 說明 |
|------|------|
| **代表性標題** | 這組新聞的代表標題 |
| **覆蓋平臺** | 哪些平臺報導了這條新聞 |
| **平臺數量** | 覆蓋了多少個平臺 |
| **是否跨平臺** | 是否為跨平臺熱點 |
| **最佳排名** | 在各平臺的最佳排名 |
| **綜合權重** | 綜合熱度評分 |
| **各平臺來源** | 各平臺的詳細信息 |

**可以調整：**

- 指定時間：如"最近一周的"
- 調整相似度閾值：如"更嚴格匹配"或"寬鬆匹配"
- 指定平臺：如"只看知乎和微博"

### Q18: 如何解析自然語言日期表達式？（推薦優先使用）

**你可以這樣問：**

- "解析'本周'是哪幾天"
- "最近7天對應的日期範圍是什麼"
- "上月的日期範圍"

**為什麼需要這個工具？**

用戶經常使用"本周"、"最近7天"等自然語言表達日期，但不同的 AI 模型自行計算日期時會產生不一致的結果。此工具使用伺服器端的精確時間計算，確保所有 AI 模型獲得一致的日期範圍。

**支援的日期表達式：**

| 類型 | 中文表達 | 英文表達 |
|------|---------|---------|
| 單日 | 今天、昨天 | today, yesterday |
| 周 | 本周、上周 | this week, last week |
| 月 | 本月、上月 | this month, last month |
| 最近N天 | 最近7天、最近30天 | last 7 days, last 30 days |
| 動態 | 最近N天（任意數字） | last N days |

**使用優勢：**

- ✅ **一致性**：所有 AI 模型獲得相同的日期範圍
- ✅ **準確性**：基於伺服器端精確時間計算
- ✅ **標準化**：返回標準日期格式
- ✅ **靈活性**：支援中英文、動態天數

## 通知推送

### Q21: 如何通過 MCP 發送通知消息？

**你可以這樣問：**

- "查看當前設定了哪些通知渠道"
- "發送一條測試消息到所有渠道"
- "把這段內容推送到飛書"
- "發送今天的新聞摘要到釘釘和 Telegram"

**支援的通知渠道（9 個）：**

| 渠道 | 消息格式 | 設定來源 |
|------|---------|---------|
| **飛書** (feishu) | 純文本 | `FEISHU_WEBHOOK_URL` |
| **釘釘** (dingtalk) | Markdown | `DINGTALK_WEBHOOK_URL` |
| **企業微信** (wework) | Markdown | `WEWORK_WEBHOOK_URL` |
| **Telegram** | HTML | `TELEGRAM_BOT_TOKEN` + `TELEGRAM_CHAT_ID` |
| **Email** | HTML | `EMAIL_FROM` + `EMAIL_PASSWORD` + `EMAIL_TO` |
| **ntfy** | Markdown | `NTFY_SERVER_URL` + `NTFY_TOPIC` |
| **Bark** | Markdown | `BARK_URL` |
| **Slack** | mrkdwn | `SLACK_WEBHOOK_URL` |
| **通用 Webhook** | Markdown | `GENERIC_WEBHOOK_URL` |

**設定方式：**

- 在 `config.yaml` 的 `notification.channels` 中設定對應渠道
- 或在 `.env` 檔案中設置對應的環境變量（優先級更高）
- 兩種方式會自動合併，`.env` 中的值會覆蓋 `config.yaml` 中的值

**兩個工具：**

| 工具 | 功能 | 示例問法 |
|------|------|---------|
| `get_notification_channels` | 檢測已設定的渠道及狀態 | "查看通知渠道設定" |
| `send_notification` | 發送消息到指定或全部渠道 | "發送消息到飛書" |

**典型使用流程：**

1. 先查看渠道狀態："查看當前設定了哪些通知渠道"
2. 確認渠道可用後發送："把以下內容推送到釘釘：今日熱點摘要..."
3. 或指定多個渠道："發送到飛書和 Telegram"
4. 不指定渠道則發送到所有已設定渠道

**消息格式：**

- 工具接受 **Markdown 格式** 的消息內容
- 自動按各渠道要求轉換格式（飛書轉純文本、Telegram 轉 HTML、Slack 轉 mrkdwn 等）
- 無需手動處理格式差異

**多帳號支援：**

- 設定值中用 `;` 分隔多個 URL/Token 即可發送到多個帳號
- 例如：`FEISHU_WEBHOOK_URL=url1;url2` 會同時發送到兩個飛書群

## 💡 使用技巧

### 1. 如何讓 AI 展示全部數據而不是自動總結？

**背景**: 有時 AI 會自動總結數據，只展示部分內容，即使工具返回了完整的 50 條數據。

**如果 AI 仍然總結，你可以**:

- **方法 1 - 明確要求**: "請展示全部新聞，不要總結"
- **方法 2 - 指定數量**: "展示所有 50 條新聞"
- **方法 3 - 質疑行為**: "為什麼只展示了 15 條？我要看全部"
- **方法 4 - 提前說明**: "查詢今天的新聞，完整展示所有結果"

**注意**: AI 仍可能根據上下文調整展示方式。


### 2. 如何組合使用多個工具？

**示例：深度分析某個話題**

1. 先搜索："搜索'人工智慧'相關新聞"
2. 再分析趨勢："分析'人工智慧'的熱度趨勢"
3. 最後情感分析："分析'人工智慧'新聞的情感傾向"

**示例：追蹤某個事件**

1. 查看最新："查詢今天關於'iPhone'的新聞"
2. 查找歷史："查找上周與'iPhone'相關的歷史新聞"
3. 找相似報導："找出和'iPhone 發布會'相似的新聞"
